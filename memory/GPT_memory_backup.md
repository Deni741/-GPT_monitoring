
# 🧠 GPT Memory Backup (v1.0)

## 👤 Основна інформація
- Користувач: Ден
- Роль GPT: головний асистент, аналітик, програміст, стратег
- Пріоритет: стабільна автономна система Telegram-бота GPT_monitoring

## 🤖 Проєкт: GPT_monitoring (Telegram бот)
- Автономна система з GPT-відповідями, кнопками, GitHub-інтеграцією, Webhook
- Сервер: root@46.254.107.104
- Telegram токен: зберігається в `.env`, не публікується
- Папка: `/root/telegram-bot/`
- Основний файл запуску: `main.py`
- GitHub автосинхронізація: через webhook + `webhook_receiver.py`
- Автозапуск: `telegram_bot.service`
- Журнал: `system_log.md`

## 📦 Функціонал бота:
1. Відповіді GPT на повідомлення та команди
2. Кнопковий інтерфейс (замість /commands)
3. Автоаналіз ринку (кнопка “Шукати сигнал”)
4. Моніторинг позицій на Bybit через Read-Only API
5. Telegram-меню: Екосистеми → Монети → Дії
6. Автооновлення через GitHub → git pull → restart
7. Файл памʼяті: GPT_memory_backup.md

## 🧠 Інструкції GPT:
- Не ламати попередню логіку коду
- Усі зміни — тільки доповнення
- Вести `system_log.md` як службовий журнал
- Уникати дублювання даних і сміття
- Якщо памʼять буде втрачена — читати цей файл і відновити ключові пункти

## 📈 Крипто-бот: стратегія
- 4 агенти:
  1. Swing Trader (RRR ≥ 1:3)
  2. StatArb (арбітраж відхилень)
  3. Sentiment Scanner (новини, пампи)
  4. Macro Tracker (керування фазами ринку)
- Працюють на окремих субакаунтах Bybit
- Керування — через Telegram

## 💸 Монетизація:
